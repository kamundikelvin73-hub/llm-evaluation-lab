# Evaluator Workflow

## Step 1: Read the Prompt

Identify:

- User intent
- Required format
- Constraints
- Safety-sensitive context
- Domain-specific requirements

## Step 2: Review Each Response

Check:

- Accuracy
- Completeness
- Clarity
- Reasoning
- Safety
- Instruction following
- Unsupported claims

## Step 3: Score Using the Rubric

Apply the relevant rubric consistently.

Use numeric scores and written notes.

## Step 4: Select the Better Response

When ranking responses, choose the answer that best satisfies the prompt while minimizing risk.

## Step 5: Write Justification

A strong evaluator justification should:

- Mention specific strengths
- Identify specific weaknesses
- Explain why one response is better
- Avoid vague statements
- Include improvement suggestions

## Step 6: Document Findings

Save results in the evaluations or reports folder.

## Evaluator Principles

- Be fair and consistent
- Do not reward confident but unsupported claims
- Prioritize safety when risk is present
- Reward clarity and usefulness
- Penalize hallucinations
- Consider user intent
